## Individual Badge Explanations

| Badge | Purpose | URL |
|-------|---------|-----|
| CI/CD Pipeline | Shows GitHub Actions workflow status | Auto-generated by GitHub |
| Build Status | Shows if latest build passed/failed | shields.io with GitHub Actions |
| Node.js Version | Shows required Node.js version | Static badge |
| React | Shows React framework and version | Static badge with logo |
| Stellar | Shows blockchain platform | Static badge with logo |
| License | Shows project license (MIT) | shields.io license badge |
| Vercel | Shows deployment platform | Static badge with logo |
| PRs Welcome | Encourages contributions | Static badge |
| GitHub Stars | Shows repository stars | Dynamic GitHub badge |
| GitHub Forks | Shows repository forks | Dynamic GitHub badge |
| GitHub Issues | Shows open issues count | Dynamic GitHub badge |
| Last Commit | Shows last commit date | Dynamic GitHub badge |

## Badge Color Codes

- **Green** (`brightgreen`): Success, passing, stable
- **Yellow** (`yellow`): Warning, license
- **Blue** (`blue`, `61DAFB`): Information, frameworks
- **Purple** (`7D00FF`): Blockchain, Stellar
- **Black** (`000000`): Deployment platforms
- **Red** (`red`): Failure, errors

## Customization Tips

1. **Change Style**: Replace `flat-square` with `flat`, `plastic`, or `for-the-badge`
2. **Add More Badges**: Visit https://shields.io for more options
3. **Custom Colors**: Use hex codes like `#FF6B6B` or names like `red`
4. **Add Logos**: Use `logo=` parameter with Simple Icons names
